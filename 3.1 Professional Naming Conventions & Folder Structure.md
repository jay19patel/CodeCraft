# Professional Naming Conventions & Folder Structure

## 1. General Naming Rules

- Use **CamelCase** for class names (`AuthService` ‚úÖ, `authservice` ‚ùå).
- Use **snake\_case** for file and function names (`auth_service.py` ‚úÖ, `AuthService.py` ‚ùå).
- Prefix **interfaces** with `I` (`IUserRepository` ‚úÖ, `UserRepositoryInterface` ‚ùå).
- Suffix classes properly based on their responsibility (`AuthService`, `UserRepository`).
- Constants should be in **UPPER_CASE** (`MAX_RETRIES = 3`).
- Avoid abbreviations (`calculate_total_price()` ‚úÖ, `calcTotalPrc()` ‚ùå).
- Keep names meaningful (`fetch_user_data()` ‚úÖ, `getData()` ‚ùå).

## 4. Function & Variable Naming Rules

### Function Naming:

- Use `snake_case` for function names: `get_user_by_id()`
- Start function names with verbs that describe the action:
  - `fetch_users()` ‚úÖ
  - `calculate_discount()` ‚úÖ
  - `validate_email()` ‚úÖ
- Private/internal functions should start with an underscore: `_hash_password()`
- Use clear and descriptive names:
  - `send_verification_email()` ‚úÖ
  - `get_user_profile_data()` ‚úÖ

### Variable Naming:

- Use `snake_case` for variables:
  - `user_name = "Jay Patel"`
  - `total_amount = 500`
- Use descriptive names, avoid single-letter variables (except for loop counters):
  - `u = get_user()` ‚ùå (not clear)
  - `user = get_user()` ‚úÖ
- Boolean variables should start with `is_`, `has_`, or `can_`:
  - `is_active = True`
  - `has_permission = False`
- Use plural names for collections:
  - `user_list = []`
  - `products = {}`
